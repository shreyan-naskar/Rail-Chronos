# Rail-Chronos

Time-series forecasting experiments for Indian Railways zone-level data using Amazon Chronos.

This repository prepares raw zone CSVs into hourly series, runs Chronos forecasts from a notebook, evaluates prediction quality across multiple forecast horizons, and generates zone-wise error plots.

## What This Project Does

The workflow in this repo is:

1. Start with raw zone files inside `raw-data/`.
2. Convert each `*_target.csv` file into individual per-series CSV files under `data/<ZONE>/`.
3. Run the forecasting notebook in `main.ipynb`.
4. Save:
   - forecast outputs,
   - evaluation metrics,
   - summary CSVs,
   - zone-wise RMSE/MAE plots.

The notebook uses `amazon/chronos-t5-small` through `ChronosPipeline`.

## Data Format

The raw archive contains zone-specific files such as:

- `CR_target.csv`
- `ER_target.csv`
- `WR_target.csv`
- corresponding `*_mask1.npz` files

`make-data.py` reads each `*_target.csv`, assumes:

- each column is an individual time series,
- data is hourly,
- timestamps begin at `2024-01-01 00:00:00`.

It then writes files like:

```text
data/CR/0.csv
data/CR/1.csv
...
```

Each generated CSV has this shape:

```csv
timestamp,target
2024-01-01 00:00:00,....
2024-01-01 01:00:00,....
```

## Setup

Create an environment and install the libraries used by the notebook and preprocessing script.

```bash
pip install pandas numpy matplotlib scikit-learn jupyter
```

You will also need the package that provides `chronos.ChronosPipeline`:

```bash
pip install chronos-forecasting
```

If you plan to run on GPU, make sure your PyTorch/CUDA setup is compatible with the Chronos package in your environment.

## How To Run

### 1. Extract the raw data

Unzip `raw-data.zip` so the repository contains:

```text
raw-data/
  CR_target.csv
  ER_target.csv
  ...
```

### 2. Build the per-series dataset

```bash
python make-data.py
```

This creates the `data/` directory expected by the notebook.

### 3. Run the notebook

Open and execute:

```text
main.ipynb
```

The notebook:

- loads `amazon/chronos-t5-small`,
- uses a 97% / 3% split for context vs. test data,
- evaluates forecast horizons from 1 to 10 hours,
- saves per-zone metrics and summaries,
- generates zone-level RMSE/MAE comparison plots.

## Outputs

### `results/`

Generated by the notebook. Expected artifacts include:

- `results/metrics_all_zones.csv`
- `results/summary_all_zones.csv`
- `results/<ZONE>/metrics.csv`
- `results/<ZONE>/summary.csv`
- `results/<ZONE>/metrics.txt`
- per-series forecast CSVs and plot images

### `plots/`

Contains summary visualizations such as:

- `plots/zone_CR_rmse_mae.png`
- `plots/zone_ER_rmse_mae.png`
- `plots/zone_WR_rmse_mae.png`

These compare mean RMSE and mean MAE against forecast horizon for each railway zone.

## Evaluation Metrics

The notebook computes:

- MSE
- RMSE
- MAE
- MAPE
- SMAPE
- Bias
- R^2

Metrics are aggregated per zone and per forecast horizon.

## Summary

This project is a compact forecasting pipeline for railway zone time series:

- `make-data.py` prepares the data
- `main.ipynb` runs Chronos forecasting and evaluation
- `plots/` stores final comparison charts
